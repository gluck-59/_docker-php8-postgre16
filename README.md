Если вам интересно — есть такие мысли:

Не знаю насколько задача соответствует архитектуре настоящего проекта, но я бы не стал использовать свою базу для хранения/добычи кладров. Эта информация от государства, она обновляется не пойми как, и обновления не всегда доступны технически (виснет-глючит-падает как и все государственное).

Я бы использовал Dadata как микросервис — эти ребята специализируются как раз на подобных вещах. Почитайте описание куска апи https://dadata.ru/api/suggest/address Цены приятные, сервис существует больше 10 лет, работает быстро, саппорт человеческий.

Флоу такой:
- юзер начинает забивать свой адрес в поле поиска
- под капотом JS стучится аяксом в Дадата и получает оттуда список подходящих адресов
- под полем поиска образуется html select с подсказками
- юзер выбирает из этого селекта, к нам на сервер летит адрес-кладр-фиас итд из ответа Дадаты и используется как нам надо

И это будет гораздо дешевле и быстрее чем хранить весь этот миллиард строк у себя и заморачиваться обновлением-слиянием по крону.
